Understanding the First Step in Learning JavaScript

September 21, 2015 / kylebrosse / Leave a comment / Edit

So one of the issues I ran into while trying to learn JavaScript is what I will call an issue of Scope. Essentially, I had no idea why I was learning anything I was learning, which created this sense that I was doing everything wrong and that I wasn’t understanding key, important information. It was, to say the least, frustrating.

Since then I have learned what I consider to be the most important lesson in learning JavaScript from somewhere other than a computing background: it’s okay to not know what you are doing.

Take, for example, the primitive data types: Boolean, Null, Number, String, Undefined, and the newly introduced Symbol. When I first started to learn JavaScript, I figured that the most important thing to learn is exactly what these data types do, and what they are used for. I can’t speak for someone who comes from a computing background because I think they might have a different take on this, but for me, understanding these data types was troublesome. They way the information was presented was at the same time very confusing and completely obvious.

For example, when describing numbers, I was told in several different ways that numbers like 23, and 3.12333 were real actual numbers and that JavaScript saw them as real actual numbers and I thought “Great! I think those are real actual numbers too!” And then it moved on without telling me A) What is JavaScript going to do with real actual numbers? and B) Why wouldn’t it think those were real actual numbers?

I’ve been doing coding for a couple months, so I’m no expert, but here is my biggest piece of advice: Do not worry about it too much. That doesn’t mean don’t care about it, let your curiosity take you as far as it can, but you are probably not going to understand the importance of everything until you actually start to use them.

Do not be overly concerned with how JavaScript understands the terms, just make sure you are know which terms JavaScript can understand. This isn’t as much about understanding how the terms are you, but understanding that JavaScript is setting the terms in which you communicate.

I wish somewhere had emphasized this more clearly. You will learn from doing how to use these tools and data types and all sorts of other functions. It’s going to come if you keep plugging away at it. The important first step is to just find the language needed to communicate with JavaScript. It’s like finding the tools before learning what they do.

Gulp, Build Tools, and Questions on Best Practices

September 17, 2015 / kylebrosse / Leave a comment / Edit

So I’ve started going to classes at Austin Coding Academy. Also, I moved to Austin. But this is the blog post for that class. This is homework now.

With that in mind, let me preface this: This post has an intended audience. I am writing it to reinforce what I have learned in the class, but it will be seen by one specific person, and so it has an intended audience. On the one hand, this will force me to write more on this blog, which has essentially gone unused while I moved, but on the other hand the purpose of the blog has shifted slightly.

Alright? Alright. Cool? Cool.

So we are learning how to install the Node Package Manager and Gulp and then eslint. All sorts of fun stuff. The first major question is, of course, why?

Well, the short of it is that JavaScript has just recently been updated. It is now super smart and way better in ways that someone of my developer development can only imagine. So this new, cool, smart JavaScript is advanced, but the support on web browsers is not. Web browsers are still dumb, though I was reading online that you can download unstable, test versions of browsers like Chrome and Firefox, but I do not know if they have the ability to run cool new JavaScript.

Anyway, we have to set up our workstations to “translate” the smart JavaScript into dumb JavaScript so that the browser can run with that code. This should also, hopefully, allow you to keep up with the now ever-changing JavaScript.

Everything we actually did, console-wise and in JavaScript, is covered by the class page:

https://github.com/AustinCodingAcademy/HTMLIntermediate/blob/master-v.2/lessons/01.md

I am not a fan of re-wording or copying and pasting. I would rather spend the time doing another build set-up, or researching things that were not covered in the lesson. So the rest of the post is going to be about Building Preferences.

As someone who is new to coding, you get thrown a bunch of terms and a bunch of new gadgets and most of them, you are told, are “Best Practice.” It is overwhelming, to say the least, especially if you are someone who is naturally suspicious whenever anyone tells you something is “best.” Best for what? Better how?

So far, what I understand about “Best Practices” are that you want to use as little code, and as simple code as you can to do a task. The idea is that your code should be neat and easy to edit and understand.

Limited understanding aside, this raises questions about Gulp and other Build Tools. Are these massive downloads really the best way to solve this translation issue? Aren’t they going to build a dependency in the user, especially if the user does not fully understand what is going on? I don’t have the answers, which should be obvious, but I feel as though I am doing my part to at least raise the question. Keith Cirkel has a blog post that advocates for a build-as-you-go approach, adding only what’s needed on the npm. You can read it here:

http://blog.keithcirkel.co.uk/why-we-should-stop-using-grunt/
Anyway, I’m going to get back to doing some coding, as well as look into online resources to help me understand Gulp, Babel and any other Build Tool I could potentially use. Thanks for reading.

DIY with HTML/CSS: Album Cover

July 1, 2015July 1, 2015 / kylebrosse / Leave a comment / Edit

There’s nothing more fun when making music than a little DIY. I’ve personally been a fan of duct tape and tape recorders when it comes to putting together a song or a show. Here’s a picture of me singing into a hockey stick:

Anyway, the other day I wanted to put up an album I had finished recording, and I got to the part where I needed to upload an album cover. The problem: I don’t have an album cover. Nor do I have photoshop. I have Microsoft Paint:

Luckily for me (and you) I do have some CSS knowledge!

First off, I created my html page and a corresponding CSS page. I’ll call the CSS album_css.css because why not. In the body, I’ll start out by creating the <div> that I will fill with the picture I want for the album cover

So then I went over to album_css and set the picture into that nice little div I just made. I picked my photo off of facebook, you can upload yours from anywhere you’d like or even from your computer. This isn’t going online, so you won’t have to worry about anyone being able to see the image on their own computer.

Then I set a ‘background-image’ declaration to the url of my photo. Now, the picture I selected has a height of 960 and a width of 866, so I set the ‘height’ and ‘width’ parameters at those numbers.

This was a mistake that you’ll see me pay for later on. I’d suggest if you go this route that you make sure that your height and width set the picture to be seen on a single page without scrolling. Trust me, it’ll save you a step or two.

\

Now, I’ve  got that sweet blurry picture of rocking out on a guitar in front of graffiti. I must be cool, because it’s indoor graffiti. Those people probably don’t care about ventilation!  Alright, time to pick a title. Now, did I go with some obscure literary reference? Of course I went with some obscure literary reference! Nothing cooler than obscure literary references on your lo-fi noise rock album.

So I set an h1 element with the name of the album. Underneath that, I set an h2 element with the name of the band. Closed everything off, now time to head to the CSS.

Now comes the fun part. Most of the problems I’ve had with paint and other image-editing programs comes down to the words on the image. They all tend to look sloppy and childish. This is where CSS skill comes in handy. Strangely enough, I have more control over positioning and style through code than with point and click.

Because no-one’s going to see this image online, I’m going to forgo ems altogether and just stick with pixels for font-size. Through a little trial and error, 75px looks about right for size. Let’s add some letter spacing to really make the words seem cleaner.

Still looks ugly. The letters don’t pop, it looks cluttered, it’s hard to read. This is exactly what I want to avoid. But how can I make it look super crazy sexy cool? First off, let’s change the font to Helvetica. Nothing like a rigid, sleek font to remove some of that clutter.

Second, let’s add a pretty dramatic text-shadow declaration. With the shifting colors on the background, neither a black nor totally white font is totally legible. One solution to this is to set a text-shadow that essentially duplicates all the letters offset in the background. That way, for the letters that don’t work in black, you’ll be able to read their white counterpart. This creates a sense of claustrophobia that reflects the title itself without sacrificing any legibility.

But there’s a problem. It’s hard to pinpoint where the title ends and the band name begins. My simple solution is to flip the color scheme, so white is in front and black is the background. I also threw in some underlines to fill in the bottom of the image a bit, which you’ll see in a minute.

Now, how do I save the image? It won’t let me take it from the web browser, for the same reason you can’t right click on espn.com and make a copy of the whole site. This is the site. Like any good DIY-er, let’s see if there’s anything around I can use. I’ve got Microsoft paint and… we’ll I’ve got microsoft paint. I’ll just take a screenshot and take it from there.

Remember when I said to change the height and width of your picture to fit on one screen? We’ll here’s why. See, I couldn’t get a screenshot that had the whole image in it.

They say the secret to DIY is a never say die attitude, but I think it’s more like wishful thinking mixed with laziness. The solution should be to just adjust the sizes of the image and the font until it all fits on one screen. But no. I tell myself “well, I’ve already got it on paint, I can just copy and paste the bottom on to the top.

Anyway, I’ll save you the details, but I eventually succeeded in human-centipede-ing my image to work. I’m happy with how it turned out, and if you want to hear the album, check it out here. I’ll keep you updated on both music and the website I’m working on for the band if and when anything interesting happens.

Mental Health And Tech: Emotional Support App

July 1, 2015 / kylebrosse / Leave a comment / Edit

One of the reasons I am getting into tech comes from my work in the mental health field. It is a field filled with talented, caring people that are doing great work. But the innovations in technology are either unfocused towards mental health or we haven’t found a way to integrate the two effectively. I wanted a place to start a dialogue about this issue, and so one segment of this blog will just be theoretical applications of technology to help those with mental health issues and those that provide that help.

There are a lot of people with mental health problems out there right now that are heading towards some sort of crises. These problems may seem small right now, bad day at work or school, small car accident, minor family emergency. But these people may be suffering from schizophrenia, or an eating disorder, or depression and this small problem is about to become a big problem unless they can get some help. The issue, however, is that their support system may be limited to three options: an expensive psychiatrist or psychologist, an overworked social worker, or institutional help. Many of these people are receiving exactly the amount of psychiatric or psychological help that they can afford and may be concerned that this now minor problem isn’t worth the major bill it will produce. Institutional help usually is reserved for times of crises and most likely will take several days, as well as also being costly. Social workers are already forced to work significant hours just to make a decent living, and may need to exceed a reasonable workload to pay off student loans.

There are many uses of technology to help people come together and communicate, but so far there has not been many applications targeted at mental health. I have seen through first hand experience working at an inpatient psychiatric hospital how help during a minor conflict can spare a major breakdown. It is like putting out a campfire to save a forest fire. Right now, most of the resources for mental health deal with the forest in flames, but I have an idea that might be offer some solution.

Essentially it would be like Uber, but for emotional support. In a time of need, you can send a literal call for help which can be picked up by someone in the area. These people would be paid small sums to come and help someone out, whether it be to talk to them or distract them or even just to lend a hand with a task. I would also like to implement a system where people with mental health problems could contact others with similar problems to lend support. Many times I have seen people’s suffering alleviate when then talk to someone who understands them. I would like to find a way to use current tech to increase those moments of compassion and help.

Is this a feasible system? Maybe, probably not. I understand the vast amounts of logistical problems a system like this produces. The biggest problem overlays with the Uber model’s biggest issue, oversight. Without regulation or oversight, how can anyone assure safety and discretion. The simplest solution would be a strict list or regulations and encouraging people to meet up in public areas. I do not have all the answers to the problems that would face this service, but I have seen the volumes of work being done every day by people in the mental health field, and I’ve seen how even the most caring and motivated people can fall short of the needs of the people who depend on them. Even with some bumps and bruises along the way, this could increase the moments of human kindness that are so essential in recovery.
